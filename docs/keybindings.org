#+TITLE: gsurf Keybindings
#+OPTIONS: toc:2 num:nil

* Syntax

Bind keys under =keybinds:= in the config. A key string is an optional set of
modifier prefixes plus a [[https://gitlab.gnome.org/GNOME/gtk/-/blob/main/gdk/gdkkeysyms.h][GDK keyval name]]:

#+begin_example
"Ctrl+r"        "Alt+Left"      "Ctrl+Shift+g"      "slash"      "j"
#+end_example

Modifiers: =Ctrl= (=Control=), =Alt= (=Mod1=), =Shift=, =Super= (=Logo=/=Mod4=).
Order and case are normalized, so =ctrl+shift+g= and =Shift+Ctrl+g= match.

Values are action nicks (see =gsurf --generate-yaml-config=). Both =-= and =_=
separators are accepted (=reload-nocache= or =reload_nocache=).

* Default bindings (core table)

| Key            | Action            |
|----------------+-------------------|
| =Ctrl+r=       | reload            |
| =Ctrl+Shift+r= | reload-nocache    |
| =Escape=       | stop              |
| =Alt+Left=     | back              |
| =Alt+Right=    | forward           |
| =Ctrl+plus/minus/0= | zoom in/out/reset |
| =Ctrl+f=       | find              |
| =Ctrl+g= / =Ctrl+Shift+g= | find next/prev |
| =F11=          | toggle-fullscreen |
| =Ctrl+q=       | quit              |

Bare-letter vim navigation (=hjkl=, =gg=, =G=, =H=/=L=, =r=, =d=/=u=) is
provided by the =modal= module (enabled by default), not the core table, so it
is mode-aware and won't interfere with typing.

*Link hinting (vimium-style).* In normal mode, =f= overlays a short chord label
on every clickable element (links, buttons, inputs, ...). Labels are drawn from
=modules.modal.hint_chars= (default =asdfghjkl=) and grow from 1 to 2/3/4
characters as the number of targets increases. Type the chord to narrow the
set; as soon as one target remains it is clicked. =F= opens the chosen link in a
new view, and =Escape= cancels. Configure the keys/colors under
=modules.modal= (=hint_key=, =hint_key_newview=, =hint_chars=, =hint_bg=,
=hint_fg=).

The =tabs= module adds =Ctrl+t= / =Ctrl+w= / =Ctrl+Tab= / =Ctrl+Shift+Tab=
(new/close/next/prev), and =toggles= adds the =Ctrl+Shift+...= setting toggles.

* Actions

Navigation: =back=, =forward=, =reload=, =reload-nocache=, =stop=, =home=,
=open-prompt=, =open-new-view=. Scroll: =scroll-{up,down,left,right,top,bottom}=,
=page-{up,down}=, =half-page-{up,down}=. Zoom: =zoom-{in,out,reset}=. Find:
=find=, =find-next=, =find-prev=. Tabs: =tab-{new,close,next,prev,reopen}=.
Modal: =enter-{normal,insert}-mode=, =follow-hints=. Misc:
=toggle-fullscreen=, =copy-url=, =paste-url=, =quit=.

* Modal / vim navigation

The =modal= module (enabled by default) provides vim-style navigation. It is
*focus-aware*, so it never blocks typing:

- When an editable element is focused — a page text field, =textarea=,
  =contenteditable=, =select=, *or the address bar* — keystrokes go straight to
  it. You do *not* need to press =i= first. =Escape= blurs the field and returns
  to command context.
- When nothing editable is focused (you're reading a page), bare letters are
  commands: =hjkl= scroll, =gg=/=G= top/bottom, =H=/=L= history, =r= reload,
  =d=/=u= half-page, =f= link hints (=F= follows in a new view).
- =i= forces INSERT mode (all keys pass to the page until =Escape=) for the rare
  web app that wants raw keys without a focused field.

Focus is tracked by a small content script that reports DOM focus changes to the
browser, so the right behavior happens automatically as you move between the
page, form fields, and the address bar.
