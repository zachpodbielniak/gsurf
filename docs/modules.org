#+TITLE: gsurf Modules
#+OPTIONS: toc:2 num:nil

Modules are loadable =.so= extensions — gsurf's equivalent of surf patches. Each
is enabled under the =modules:= section of the config. See [[file:module-system.org][module-system.org]] to
write one. =gsurf --list-modules= prints the registry.

Status: *implemented* modules ship and are tested; *planned* modules are designed
(registry entry + config schema present) and follow the established pattern.

* Tier 3 — nice-to-have

=spacesearch=, =inspector=, =dark_mode=, =status_bar=, =geolocation=,
=session=, and =proxy_switch= are implemented. =webgl_toggle= and
=dlconsole= are intentionally folded into =toggles= and =downloads=
respectively (they were redundant catalog entries).

All 29 modules build with =make= (the =mcp= module requires =MCP=1=).

* AI modules (deferred)

A future =AI=1= build flag (mirroring gst's =ai-glib= path) would add AI-powered
modules such as page-summary and chat. The architecture reserves room for it
(hook interfaces are already general enough); no AI modules are built yet.

* Per-module documentation

Detailed docs live under =docs/modules/<name>.org= as each module lands. The
=search_engines= and =mcp= modules are the reference implementations.
